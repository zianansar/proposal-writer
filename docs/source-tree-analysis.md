# Source Tree Analysis

## Directory Structure

```
Upwork Researcher/
├── .agent/              # Agent configuration and state
├── .git/                # Git repository metadata
├── _bmad/               # BMAD Framework Core
│   ├── _config/         # Agent and system configurations
│   ├── bmm/             # Build-Manage Module (Agents, Workflows)
│   └── core/            # Core Workflow Engine
├── _bmad-output/        # Generated Project Artifacts (Research/Implementation)
│   ├── brainstorming/   # Brainstorming session logs
│   ├── implementation-artifacts/ # Guides, templates, and frameworks
│   └── planning-artifacts/ # Research reports, PRDs, UX specs
├── docs/                # Project Documentation (Auto-generated)
└── CLAUDE.md            # Project Overview and Agent Guide
```

## Critical Directories

| Directory       | Purpose              | Key Content                                                       |
| :-------------- | :------------------- | :---------------------------------------------------------------- |
| `_bmad/`        | **Framework Engine** | Contains the intelligence of the system (agents, workflows).      |
| `_bmad-output/` | **Project Outputs**  | Stores all value generated by the agents (research, plans, code). |
| `docs/`         | **Knowledge Base**   | Stores reference documentation (like this file).                  |

## Entry Points

*   **Framework Entry**: `_bmad/core/tasks/workflow.xml` (The engine that runs everything)
*   **User Entry**: `CLAUDE.md` (Provides slash commands to start agents)

## Integration Points

As a Monolith/Infrastructure project, everything is integrated via the **BMAD Framework**. Agents in `_bmad/bmm/agents` utilize workflows in `_bmad/bmm/workflows` to generate output in `_bmad-output`.
